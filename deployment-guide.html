Project Objective

Lambda Execution Flow

create_ticket_lambda

  • Receives POST request
  • Generates UUID Ticket ID
  • Assigns Category
  • Stores Ticket in DynamoDB
  • Sends SNS Notification

update_ticket_lambda

  • Receives PUT request
  • Updates Ticket Status
  • Sends SNS Email Notification

dashboard_stats_lambda

  • Scans DynamoDB
  • Calculates Dashboard Metrics

Resume Project Description

Designed and implemented a fully serverless Enterprise Support Ticket System using AWS services including Cognito, API Gateway, Lambda, DynamoDB, SNS, S3, CloudFront, and Route53. Implemented secure authentication, ticket management workflows, admin dashboard analytics, SNS email notifications, and serverless REST APIs while following cloud-native architecture principles.

Project Explanation for Interviews

The project is a serverless support ticket management system hosted on Amazon S3 and CloudFront. Users authenticate through Amazon Cognito and create support tickets. API Gateway routes requests to Lambda functions which process business logic and store data in DynamoDB. SNS is used to send email notifications for high-priority tickets and status changes. An Admin Dashboard allows support engineers to manage tickets, update statuses, view analytics, and monitor ticket lifecycle activities.
